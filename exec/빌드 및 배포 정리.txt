3) 배포 시 특이사항 기재
Jenkins를 활용해서 EC2에 배포했습니다. 

/home/ubuntu/aicheck/db/mysql/* 해당 경로에 exec/db/mysql/* 경로에 존재하는 sql 및 sh 파일이 존재 해야 합니다.
/home/ubuntu/aicheck/db/mongo/* 해당 경로에 exec/db/mongo/* 경로에 존재하는 sql 및 sh 파일이 존재 해야 합니다.
/home/ubuntu/custom-jenkins/ 해당 경로에 Dockerfile 이 존재 해야 합니다.
/home/ubuntu/custom-jenkins/ 해당 경로에 run-jenkins.sh 이 존재해야 합니다.

/home/ubuntu/custom-jenkins 경로로 이동하여 ./run-jenkins.sh 명령어를 실행하여 jenkins 컨테이너를 실행합니다.

---
jenkins Dockerfile
/home/ubuntu/custom-jenkins/Dockerfile

FROM jenkins/jenkins:lts

USER root

# 호스트와 동일한 GID (122)에 맞춰 docker 그룹 생성
RUN groupadd -g 122 docker && \
    usermod -aG docker jenkins

# 기본 패키지 설치
RUN apt-get update && \
    apt-get install -y curl unzip gnupg lsb-release awscli docker.io && \
    rm -rf /var/lib/apt/lists/*

# docker compose 디렉토리 생성 후 바이너리 설치
RUN mkdir -p /usr/local/lib/docker/cli-plugins && \
    curl -SL https://github.com/docker/compose/releases/download/v2.27.0/docker-compose-linux-x86_64 \
      -o /usr/local/lib/docker/cli-plugins/docker-compose && \
    chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

# docker 그룹에 jenkins 사용자 추가
RUN usermod -aG docker jenkins

USER jenkins

---
젠킨스 컨테이너 실행 .sh 파일
/home/ubuntu/custom-jenkins/run-jenkins.sh

#!/bin/bash

# 기존 jenkins 컨테이너 종료 및 삭제
docker stop jenkins 2>/dev/null || true
docker rm jenkins 2>/dev/null || true

# 이미지 빌드
docker build -t custom-jenkins -f Dockerfile .

# 컨테이너 실행
docker run -d \
  --name jenkins \
  -p 9090:8080 \
  -v /home/ubuntu/jenkins-data:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -v /home/ubuntu/aicheck/env:/home/ubuntu/aicheck/env \
  -v /home/ubuntu/aicheck/db/mysql:/home/ubuntu/aicheck/db/mysql \
  -v /home/ubuntu/aicheck/db/mongo:/home/ubuntu/aicheck/db/mongo \
  custom-jenkins

  4) DB 접속 정보 등 프로젝트(ERD)에 활용되는 주요 계정 및 프로퍼티가 정의된 파일 목록
  exec/.env 에 전부 적혀있습니다.
